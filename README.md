# Moscow Real Estate Intelligence

[Русская версия](README_RU.md)

An end-to-end data analytics and machine learning project focused on the Moscow residential real estate market from 2020 to 2026.

The project combines Python-based data processing, exploratory data analysis, machine learning models and an interactive Power BI dashboard to analyze sales prices, rental prices and predicted rental yields across Moscow districts.

---

## Project Overview

This project analyzes Moscow real estate listings from three perspectives:

* **Sales market analysis** — secondary market and new build prices by district, room count and distance to the city center.
* **Rental market analysis** — rental price dynamics and rent drivers such as rooms, renovation, furnishing and pet policy.
* **Investment analysis** — ML-based estimation of predicted sale prices, predicted monthly rent and gross rental yield.
* **Power BI dashboard** — interactive business intelligence dashboard for exploring key market indicators.

The goal of this project is to demonstrate a complete analytics workflow: from raw data to clean datasets, machine learning models and a final dashboard suitable for portfolio presentation.

## Python Pipeline

The project is organized as a sequential pipeline.

### 1. Data Quality Check

```bash
python scripts/01_check_data.py
```

Checks source datasets, column structure, missing values and basic dataset statistics.

### 2. Exploratory Data Analysis

```bash
python scripts/02_basic_eda.py
```

Generates exploratory charts and summary tables for sales, rental and new build listings.

### 3. Data Cleaning and Feature Engineering

```bash
python scripts/03_prepare_clean_data.py
```

Creates cleaned datasets and engineered features, including:

* listing year, month and quarter
* price in million RUB
* rent in thousand RUB
* price and rent per square meter
* building age
* floor position indicators
* distance-to-center groups
* room groups
* furnishing and pet policy labels
* market type labels

### 4. Power BI Table Preparation

```bash
python scripts/04_prepare_powerbi_tables.py
```

Creates analytical CSV tables used in the Power BI dashboard.

### 5. Sales Price Model

```bash
python scripts/05_train_price_model.py
```

Trains machine learning models to predict apartment sale prices.

Models tested:

* baseline median model
* linear regression
* random forest
* gradient boosting

### 6. Rental Price Model

```bash
python scripts/06_train_rent_model.py
```

Trains machine learning models to predict monthly rental prices.

Models tested:

* baseline median model
* linear regression
* random forest
* gradient boosting

### 7. Investment Prediction Tables

```bash
python scripts/07_prepare_investment_predictions.py
```

Combines predicted sale prices and predicted rental prices to estimate gross rental yield and classify listings by investment score.

Investment score groups:

* Low Yield
* Medium Yield
* High Yield

## Machine Learning Approach

Two separate supervised regression tasks were used:

1. **Sale price prediction**

   * Target: `price_rub`
   * Input features include apartment area, rooms, floor characteristics, district, metro station, distance to center and market type.

2. **Monthly rent prediction**

   * Target: `monthly_rent_rub`
   * Input features include apartment area, rooms, floor characteristics, district, metro station, distance to center, building features and rental conditions.

The predicted values are then combined to estimate gross rental yield:

```text
Gross rental yield = predicted annual rent / predicted sale price
```

---

## Important Limitations

The investment analysis uses **gross rental yield**.

This metric does not include:

* taxes
* vacancy periods
* maintenance costs
* mortgage payments
* transaction costs
* renovation costs
* property management expenses

Model validation currently uses a random train-test split. For real forecasting use cases, a time-based validation split would be more appropriate because the real estate market changes over time.

Therefore, the investment score should be interpreted as a comparative analytical indicator, not as financial advice.

## How to Run the Project

### 1. Clone the repository

```bash
git clone https://github.com/k-tsoy/moscow-real-estate-intelligence.git
cd moscow-real-estate-intelligence
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

macOS / Linux:

```bash
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Download the dataset

Download the dataset from Kaggle and place the raw CSV files into:

```text
data/raw/
```

Required files:

```text
district_prices_monthly.csv
metro_stations.csv
new_builds.csv
rentals.csv
secondary_market.csv
```

### 5. Run the pipeline

```bash
python scripts/01_check_data.py
python scripts/02_basic_eda.py
python scripts/03_prepare_clean_data.py
python scripts/04_prepare_powerbi_tables.py
python scripts/05_train_price_model.py
python scripts/06_train_rent_model.py
python scripts/07_prepare_investment_predictions.py
```

### 6. Open the Power BI dashboard

Open the file:

```text
powerbi/moscow_real_estate_dashboard.pbix
```

Then click:

```text
Refresh
```
